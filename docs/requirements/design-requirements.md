# Madoka Launch Engine - デザイン要件

## 1. 美的方向性

### 1.1 トーン & ムード
- **方向性**: Professional Luxury - 洗練されたプロフェッショナルツール
- **ムード**: 知的・信頼感・高級感
- **差別化**: マーケティングツールにありがちな派手さではなく、落ち着いた上質感。使うたびに信頼できるパートナーのような存在感。
- **参照イメージ**: Notion的なクリーンさ + 高級ブランドサイトの余白感

### 1.2 デザイン原則
1. **Clarity First** - 情報の優先順位が一目でわかる
2. **Breathing Space** - 余白を恐れない。各要素に呼吸の空間を
3. **Professional Warmth** - 冷たすぎず、温かみのあるプロフェッショナル感
4. **Action-Oriented** - 次に何をすべきかが常に明確

## 3. UIコンポーネント方針

### 3.1 技術スタック
| 項目 | 選定 |
|------|------|
| UIライブラリ | shadcn/ui |
| スタイリング | Tailwind CSS |
| アニメーション | motion/react (framer-motion) |
| アイコン | Lucide React |
| チャート | Recharts |
| ユーティリティ | cn (class-variance-authority) |

### 3.2 コンポーネントルール
- ボタン: 角丸 8px、ホバーアニメーション 150ms
- カード: 角丸 12px、微細なシャドウ、ボーダー 1px
- モーダル: バックドロップブラー、中央配置
- トースト: 右下固定、3秒自動消去
- ローディング: スケルトンスクリーン（スピナーは使わない）

### 3.3 禁止パターン
- グラデーション背景（安っぽくなるため）
- ネオンカラー/蛍光色
- 過剰なドロップシャドウ
- アニメーションの多用（200ms以上のアニメーション禁止）
- 過剰な装飾要素

## 4. レスポンシブ方針

### 4.1 ブレークポイント
| 名称 | サイズ | 用途 |
|------|--------|------|
| sm | 640px | スマホ横 |
| md | 768px | タブレット |
| lg | 1024px | ノートPC |
| xl | 1280px | デスクトップ |
| 2xl | 1536px | ワイドモニター |

### 4.2 PC-First設計
- **メインターゲット**: デスクトップ（1280px以上）
- **スマホ対応必須画面**: ダッシュボード日次入力、配信文プレビュー
- **スマホ非対応可**: ファネル設計、スライド生成、LP編集

### 4.3 レイアウト
- サイドバー: 固定幅 256px（折りたたみ可能）
- メインエリア: 残り幅全体
- 最大コンテンツ幅: 1200px（中央配置）

## 5. 特殊UI要件

### 5.1 AI対話UI
- チャットバブル形式（ユーザー: 右寄せ、AI: 左寄せ）
- AIレスポンスのストリーミング表示
- 提案カード: AIの提案をカード形式で表示、選択可能
- コンテキスト表示: 何のナレッジを参照したか表示

### 5.2 ファネルビジュアライザー
- ファネル形状の図で各ステップを表示
- 各ステップ間にCVRを表示
- ステップのドラッグ&ドロップによる並べ替え
- ステップの色でステータスを表現（未設定/生成中/完了）

### 5.3 コピースコアリングUI
- 100点満点のリングチャート
- 各評価項目のレーダーチャート
- 弱い項目にはワンクリック改善ボタン

### 5.4 ダッシュボードUI
- ファネル図にリアルタイム数値をオーバーレイ
- 日別推移の折れ線グラフ
- CVRの色分け（目標以上: 緑、やや下: 黄、大幅下: 赤）
- ボトルネック箇所のパルスアニメーション

### 5.5 コンテンツ編集UI
- 左ペイン: セクション一覧（ナビゲーション）
- 右ペイン: エディタ（マークダウン対応）
- 下部パネル: AI提案/対話修正（折りたたみ可能）
- フローティングボタン: ワンクリックコピー

### 5.6 配信カレンダーUI
- 月表示/週表示の切替
- 配信種別ごとの色分け
- ドラッグで日程変更
- 配信内容のポップオーバープレビュー
