# Madoka Launch Engine - 非機能要件

## 1. パフォーマンス

| 項目 | 要件 |
|------|------|
| ページ読み込み | 初回: 3秒以内、遷移: 1秒以内 |
| AI生成レスポンス | ストリーミング表示、完了まで30秒以内（LP1セクション） |
| LP全セクション一括生成 | 5分以内 |
| 配信文一括生成（10通） | 3分以内 |
| ダッシュボードデータ表示 | 2秒以内 |
| CSV取込処理 | 1,000行: 5秒以内 |
| コピースコアリング | 5秒以内 |
| Canva API連携（スライド生成） | 200枚: 10分以内 |

## 2. セキュリティ

| 項目 | 要件 |
|------|------|
| 認証 | Supabase Auth（メール+パスワード） |
| 認可 | シングルユーザー（円香のみ）、将来的なマルチユーザー拡張を考慮 |
| データ暗号化 | 通信: TLS 1.3、保存: AES-256 |
| APIキー管理 | 環境変数管理、クライアントに露出させない |
| クライアントデータ分離 | プロジェクト単位でデータを論理分離 |
| セッション管理 | JWT、24時間で有効期限切れ |
| CORS | 許可オリジンのみ |

## 3. 可用性

| 項目 | 要件 |
|------|------|
| 稼働率 | 99.5%以上（Vercel + Supabase SLA準拠） |
| バックアップ | 日次自動バックアップ（Supabase） |
| データ復旧 | 7日以内のデータ復旧可能 |
| エラーハンドリング | AI生成失敗時のリトライ（3回まで自動） |

## 4. スケーラビリティ

| 項目 | 要件 |
|------|------|
| 同時プロジェクト数 | 50プロジェクト以上 |
| ナレッジベースサイズ | 1,000素材以上 |
| お客様の声登録数 | 500件以上 |
| ローンチ数値データ | 365日分/プロジェクト |
| 生成コンテンツ保存 | 無制限（Supabase Storage） |

## 5. ユーザビリティ

| 項目 | 要件 |
|------|------|
| レスポンシブ | PC優先、スマホ対応（ダッシュボード入力用） |
| ワンクリックコピー | 全テキストコンテンツにコピーボタン |
| 自動保存 | 編集内容の自動保存（5秒間隔） |
| Undo/Redo | コンテンツ編集時のUndo/Redo対応 |
| ダークモード | 対応 |
| AI対話UI | チャット形式のインターフェース |
| ドラッグ&ドロップ | ファネル設計、カレンダー操作 |

## 6. 信頼性

| 項目 | 要件 |
|------|------|
| AI生成品質 | コピースコア80点以上を初回生成で達成率70%以上 |
| データ整合性 | トランザクション管理、データ不整合の防止 |
| エラー通知 | 生成失敗・API障害時にアプリ内通知 |
| 入力バリデーション | 必須項目チェック、型チェック |

## 7. 保守性

| 項目 | 要件 |
|------|------|
| コードベース | TypeScript strict mode |
| テスト | ユニットテスト70%以上カバレッジ |
| ログ | 構造化ログ（AI生成ログ含む） |
| 監視 | Vercel Analytics + エラー監視 |
| プロンプト管理 | プロンプトテンプレートのバージョン管理 |

|------|------|
| モデル | Claude API（claude-sonnet-4-20250514 以上） |
