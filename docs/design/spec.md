## 2. 画面仕様

### 2.1 ダッシュボード（ホーム）

**URL**: `/dashboard`

**目的**: アプリ起動時の全体概要。アクティブプロジェクトの状況を一覧で把握。

| 要素 | 仕様 |
|------|------|
| アクティブプロジェクトカード | 最大6件、プロジェクト名/タイプ/進捗/直近KPI |
| クイックアクション | 「新規プロジェクト作成」「ナレッジ追加」 |
| ローンチ中アラート | ローンチ中プロジェクトの注目指標をハイライト |
| 最近の生成物 | 直近で生成/編集したコンテンツ一覧 |

AIがチャット形式で質問し、回答からペルソナを構築:

```
│  │                                         │ │
│  │ [メッセージ入力...              ] [送信]│ │
│  └─────────────────────────────────────────┘ │
│                                              │
│  ┌─ ペルソナプレビュー（リアルタイム更新）─┐ │
│  │ 基本属性: 30〜40代女性、起業初期...     │ │
│  │ 悩みTOP5: ...                           │ │
│  │ 理想の未来: ...                         │ │
│  │ 失敗経験: ...                           │ │
│  │ 購買不安: ...                           │ │
│  └─────────────────────────────────────────┘ │
│                                              │
│            [← 戻る]      [次へ →]           │
└─────────────────────────────────────────────┘
```

**AI壁打ちの質問シーケンス:**

1. 基本属性（年齢/性別/職業/年収）
2. 悩み・課題TOP5
3. 理想の未来（ビフォーアフター）
4. 今まで試した解決策と失敗理由
5. 購買時の不安・反論
6. 情報収集チャネル
7. 使う言葉・口癖（任意）

各回答後、右側のペルソナプレビューがリアルタイムで更新される。

### 2.3 プロジェクト詳細画面

**URL**: `/projects/[id]`

上部にプロジェクト情報サマリー、左サイドにモジュール別タブナビゲーション。

#### 2.3.1 概要タブ

| 要素 | 説明 |
|------|------|
| プロジェクト情報カード | タイプ/商品名/価格/ステータス |
| ペルソナサマリー | 主要属性/悩み/理想の未来 |
| オファーサマリー | 価格/特典/保証 |
| ファネル概要 | 選択したファネルの図 |
| 生成進捗 | 各コンテンツの生成状態（未生成/生成中/完了） |
| クイックアクション | 「LP生成」「配信生成」「ダッシュボード」 |

## 3. AI生成エンジン仕様

### 3.1 プロンプトアーキテクチャ

```
System Prompt
├── マーケティングフレームワーク定義
├── コピーライティング原則（3つのNOT, GDT等）
├── 品質基準定義
└── トーン/スタイル指示

User Prompt (動的構築)
├── 商品情報（プロジェクトから）
├── ペルソナ情報（プロジェクトから）
├── オファー情報（プロジェクトから）
├── RAG結果（ナレッジベースから類似素材）
├── お客様の声（実績DBから関連実績）
├── トンマナ指示（プロジェクトから）
├── 生成対象の指定（LP種別/セクション/配信タイプ）
└── 追加指示（ユーザーの修正指示）
```

### 3.2 生成フロー

```
1. コンテキスト収集
   ├── プロジェクト情報の取得
   ├── RAG検索（関連ナレッジ上位5件）
   └── 関連実績の取得（タグマッチ上位5件）

2. フレームワーク選択
   ├── LP種別 × ペルソナ認知レベル → 最適なフレームワーク
   └── OATHフォーミュラでターゲットの認知段階を判定

3. プロンプト構築
   └── テンプレート + 動的データ → 最終プロンプト

4. 生成 (Claude API)
   ├── ストリーミングで逐次表示
   └── セクション単位で区切って出力

5. コピースコアリング
   ├── 10項目 × 10点 = 100点満点
   ├── 80点以上 → そのまま出力
   └── 80点未満 → 弱い項目を特定して自動改善ループ（最大2回）

6. ユーザーレビュー
   ├── セクション単位で確認
   ├── AI対話で修正指示
   └── 確定 → 保存
```

### 3.3 生成単位

| 対象 | 生成単位 | 理由 |
|------|---------|------|
| LP | セクション単位 | 1回のAPI呼び出しで全セクションは長すぎる。セクション単位で品質を担保 |
| 配信文 | 1通ずつ | 前の配信との一貫性を保つため順次生成 |
| セミナースライド | パート単位（30〜50枚） | 構成の一貫性を保ちつつ分割 |
| VSL台本 | セクション単位 | LP同様 |
| SNS投稿 | 1投稿ずつ | 個別最適化 |

### 3.4 コピースコアリングロジック

```typescript
interface CopyScore {
  headline: number;       // ヘッドライン（ベネフィット・好奇心）
  problemStatement: number; // 問題提起（具体性・共感度）
  logicFlow: number;      // 論理構成（フレームワーク適合度）
  socialProof: number;    // 社会的証明（実績の量と質）
  offer: number;          // オファー（価値の伝わりやすさ）
  cta: number;            // CTA（明確性・緊急性）
  objectionHandling: number; // 反論処理（不安への対応）
  readability: number;    // 読みやすさ（文の長さ・リズム）
  targetFit: number;      // ターゲット適合度
  uniqueness: number;     // 独自性
  total: number;          // 合計 /100
}
```
